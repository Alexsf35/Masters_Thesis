## 📘 Project Overview

**Genome-scale metabolic models (GSMMs)** are essential tools for simulating cellular metabolism and exploring biological hypotheses. However, their complexity often makes it difficult to **interpret simulation outputs**, such as flux changes or the identification of essential genes.  

This project aims to **enhance the interpretability of GSMMs** through the **integration of biological knowledge using Artificial Intelligence (AI)**. By combining **large language models (LLMs)** with **biological knowledge graphs (KGs)**, the goal is to build a system that links GSMM entities to information from **scientific literature (PubMed)** and **protein interaction databases (STRING)**.  

Integration will be achieved using **BioCypher**, while reasoning over the graph will be handled by **BioChatter**, enabling **natural language queries** over the integrated data.

---

## 🎯 Objectives

1. **Review the literature** and prepare the pre-thesis.  
2. **Develop an integration architecture** connecting GSMM entities with external biological data (PubMed, STRING).  
3. **Build the integrated knowledge graph** using BioCypher and implement a reasoning interface powered by LLMs via BioChatter.  
4. **Validate the developed approach** using human GSMMs, explaining simulation results (e.g., gene knockouts, flux predictions) through AI-assisted interpretation.  
5. **Deliver reproducible pipelines** and evaluation scripts demonstrating interpretability gains and biological relevance.

---

## 🧠 Methodology

- **BioCypher:** Framework for unifying and structuring biomedical data.  
- **BioChatter:** LLM-based reasoning interface for interacting with structured biomedical knowledge.  
- **STRING:** Database of protein–protein interactions supported by experimental and text-mining evidence.  
- **PubMed API:** Used to retrieve and summarize relevant biomedical literature.  
- **Human GSMMs:** Models such as **Recon3D** or **Human-GEM** will be used for validation and case studies.
